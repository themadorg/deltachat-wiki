# اطلاعات توسعه‌دهندگان

ساخت بات دلتا چت خودتان آسان است.
ادامه مطلب را بخوانید برای برخی اطلاعات پایه و شاید نگاهی به [بات‌های نمونه ما](https://deltachat-bot.github.io/public-bots/) بیندازید.

## API

بات‌های دلتا چت از [هسته chatmail](https://github.com/chatmail/core) استفاده می‌کنند، موتوری که بیشتر کارها را انجام می‌دهد.

کار شما به عنوان نویسنده بات، استفاده از خروجی آن موتور و دادن دستورالعمل به آن است. شما نیازی به تجزیه هیچ پیام ایمیلی، بررسی اتصالات یا نگرانی درباره رمزگذاری ندارید، همه اینها کار موتور است.

موتور هسته می‌تواند به صورت بومی از Rust و از طریق [API JSON-RPC](https://jsonrpc.delta.chat/) از هر زبان دیگری استفاده شود.
(لطفاً توجه داشته باشید که API هنوز ممکن است تغییر کند)

اگر نیاز به نقطه شروع برای بات خود دارید، به [بات echo](https://github.com/deltachat-bot/echo) نگاه کنید، که نمونه‌هایی برای شروع در چندین زبان دارد.

(اگر زبان مورد علاقه شما در اینجا نیست، لطفاً نوشتن اتصالات به API برای آن را در نظر بگیرید!)


## پیش‌زمینه

موتور هسته دلتا چت تمام اتصالات شبکه را مدیریت می‌کند و فعالیت‌ها را زیر نظر می‌گیرد.
همچنین راه‌هایی برای تعامل با داده‌های شناخته شده (مثلاً خواندن پیام‌ها) و همچنین ایجاد داده‌های جدید (مثلاً مدیریت مخاطبین، ارسال پیام‌ها) ارائه می‌دهد.

شروع موتور هسته دلتا چت به معنای شروع ریسمان‌هایی است که به سرور ایمیل شما متصل می‌شوند و روی کارهایی مانند نظارت بر پیام‌های جدید، دریافت پیام‌ها و ارسال پیام‌ها کار می‌کنند.
هر فعالیتی که موتور هسته ارزش دانستن آن را داشته باشد، از طریق رویدادها پخش می‌شود.
برای اینکه از آنچه اتفاق می‌افتد باخبر باشید، بات شما باید رویدادهایی را که می‌خواهد مدیریت کند، گوش دهد.

#### رویدادها

اینجا [لیست همه رویدادهای موجود](./events) با توضیحات معنای آن‌ها آمده است.

بیشتر رویدادها دارای بارداده یک یا دو آرگومان هستند که داده‌های مرتبط را شامل می‌شوند. این ممکن است یک پیام ورودی باشد، اما ممکن است یک عدد باشد که پیشرفت یک عملیات را نشان می‌دهد.

یک مثال مفید برای یک رویداد و بارداده آن [`DC_EVENT_INCOMING_MSG`](./events) است، رویدادی که هنگام رسیدن پیام جدید برای یک چت موجود منتشر می‌شود.
بارداده آن شناسه چتی است که پیام برای آن رسیده و شناسه خود پیام.
اگر آرگومان بارداده 0 باشد، خطا رخ داده است. خطاها از طریق رویدادهای مختلف پخش می‌شوند.
برای دسترسی به چت و پیام واقعی، کد شما باید آن‌ها را با فراخوانی API از پایگاه داده دریافت کند.

> **توجه**: فریم‌ورک‌های بات سطح بالا `DC_EVENT_INCOMING_MSG` را برای شما مدیریت می‌کنند و داده‌های پیام ورودی را در اختیار شما قرار می‌دهند، این فقط برای توضیح پیش‌زمینه ذکر شده است.
