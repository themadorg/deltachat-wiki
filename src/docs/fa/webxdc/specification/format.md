## فرمت فایل کانتینر Webxdc

- یک برنامه Webxdc یک **فایل ZIP** با پسوند `.xdc` است

- فایل ZIP باید از متدهای فشرده‌سازی پیش‌فرض مطابق RFC 1950 استفاده کند،
  یعنی «Deflate» یا «Store»

- فایل ZIP حداقل باید شامل فایل `index.html` باشد

- فایل ZIP ممکن است شامل فایل‌های `manifest.toml` و `icon.png` یا
  `icon.jpg` باشد


### فایل manifest.toml

اگر فایل ZIP شامل `manifest.toml` در پوشه ریشه خود باشد،
اطلاعات پایه زیر باید از آن خوانده شود:

```toml
name = "My App Name"
source_code_url = "https://example.org/orga/repo"
```

- `name` - نام برنامه webxdc.
  اگر نامی تنظیم نشده یا manifest وجود نداشته باشد، نام فایل به عنوان نام webxdc استفاده می‌شود.

- `source_code_url` - URL اختیاری که کد منبع webxdc و شاید اطلاعات دیگر در آن قابل دسترسی است.
  پیاده‌کنندگان پیام‌رسان ممکن است URL را از طریق منوی «راهنما» در پنجره webxdc در دسترس قرار دهند.


### فایل‌های آیکون

اگر ریشه ZIP شامل `icon.png` یا `icon.jpg` باشد،
این فایل‌ها به عنوان آیکون webxdc استفاده می‌شوند.
آیکون باید مربعی با عرض/ارتفاع مناسب باشد،
معمولاً بین 128 × 128 و 512 × 512 پیکسل.
گوشه‌های گرد، برش دایره‌ای و غیره توسط پیاده‌سازی‌ها در صورت نیاز اضافه خواهند شد؛
بنابراین حاشیه یا شکل به آیکون اضافه نکنید.
اگر آیکونی تنظیم نشده باشد، آیکون پیش‌فرض استفاده خواهد شد.
