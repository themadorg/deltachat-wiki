# Информация для разработчиков

Создать собственного бота для Delta Chat очень просто.
Ознакомьтесь с основной информацией ниже и, возможно, взгляните на наши [примеры ботов](https://deltachat-bot.github.io/public-bots/).

## API

Боты Delta Chat используют [ядро chatmail](https://github.com/chatmail/core) — движок, выполняющий большую часть работы.

Ваша задача как автора бота — использовать выходные данные этого движка и давать ему инструкции. Вам не придется парсить электронные письма, проверять соединения или заботиться о шифровании — всё это работа движка.

Ядро можно использовать нативно из Rust и через [JSON-RPC API](https://jsonrpc.delta.chat/) из любого другого языка.
(Обратите внимание, что API всё ещё может меняться).

Если вам нужна отправная точка для вашего бота, посмотрите на [эхо-бота](https://github.com/deltachat-bot/echo), в котором есть примеры для быстрого старта на нескольких языках.

(Если для вашего любимого языка здесь нет примера, подумайте о написании привязок (bindings) к API для него!)

## Техническая часть

Ядро Delta Chat управляет всеми сетевыми соединениями и следит за активностью.
Оно также предоставляет способы взаимодействия с известными данными (например, чтение сообщений) и создания новых данных (например, управление контактами, отправка сообщений).

Запуск ядра Delta Chat означает запуск потоков, которые подключаются к вашему почтовому серверу и выполняют такие задачи, как отслеживание новых сообщений, получение сообщений и их отправка.
Любая активность, которую ядро считает заслуживающей внимания, транслируется через события.
Чтобы быть в курсе происходящего, ваш бот должен слушать события, которые он хочет обрабатывать.

#### События

Вот [список всех доступных событий](./events) с описанием их значений.

Большинство событий несут полезную нагрузку из одного или двух аргументов, содержащих связанные данные. Это может быть входящее сообщение, а может быть и число, означающее прогресс операции.

Одним из полезных примеров события и его полезной нагрузки является [`DC_EVENT_INCOMING_MSG`](./events) — событие, возникающее при поступлении нового сообщения в существующий чат.
Его полезная нагрузка — это ID чата, в который пришло сообщение, и ID самого сообщения.
Если аргумент полезной нагрузки равен 0, значит произошла ошибка. Ошибки транслируются через другие события.
Чтобы получить доступ к самому чату и сообщению, вашему коду нужно будет запросить их из базы данных, вызвав API.

> **Примечание**: высокоуровневые фреймворки для ботов обрабатывают `DC_EVENT_INCOMING_MSG` за вас и предоставляют данные входящего сообщения. Это упомянуто здесь только для объяснения того, как всё устроено «под капотом».
