# Особенности реализации

На этой странице документируется необычное поведение реализаций протоколов в maddy.
Некоторые из этих проблем нарушают стандарты, некоторые — нет, но всё равно могут мешать совместимости.

## SMTP

- Поле `for` никогда не включается в поле заголовка `Received`.

  Это разрешено стандартом [RFC 2821].

## IMAP

### `sql`

- Флаг `\Recent` сбрасывается не во всех случаях.

  Это _не нарушает_ [RFC 3501]. Клиенты, полагающиеся на него, будут работать (гораздо) менее эффективно.

[RFC 2821]: https://tools.ietf.org/html/rfc2821
[RFC 3501]: https://tools.ietf.org/html/rfc3501
