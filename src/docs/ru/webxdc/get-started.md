# Начало работы

Webxdc внедряет веб-приложения в чаты мессенджеров,
используя простой формат файлов, содержащий HTML5, CSS, JavaScript и другие ресурсы.
Вся аутентификация, управление идентификацией, социальный поиск и передача сообщений
передаются мессенджеру-хосту, который запускает файл контейнера приложения webxdc
и ретранслирует сообщения об обновлении приложения между пользователями.
Это позволяет каждому приложению наследовать возможности работы в автономном режиме (offline-first) 
и сквозного шифрования (end-to-end encryption), реализованные в мессенджере.

## Простой пример

Мессенджеры [Delta Chat](https://delta.chat) (на базе электронной почты) 
и [Cheogram](https://cheogram.com) (на базе XMPP) 
поддерживают [приложения webxdc](https://webxdc.org/apps), которые работают в обоих мессенджерах без каких-либо изменений.

Следующий файл `index.html` представляет собой законченное приложение webxdc с полем ввода, отображаемым у всех участников. Данные, введенные в поле, доставляются всем участникам чата.

```html
<!DOCTYPE html>
<html>
  <head>
    <meta charset="utf-8"/>
    <script src="webxdc.js"></script>
  </head>
  <body>
    <input id="input" type="text"/>
    <a href="" onclick="sendMsg(); return false;">Отправить</a>
    <p id="output"></p>
    <script>
      function sendMsg() {
        msg = document.getElementById("input").value;
        window.webxdc.sendUpdate({payload: msg}, 'Кто-то напечатал "'+msg+'".');
      }
    
      function receiveUpdate(update) {
        document.getElementById('output').innerHTML += update.payload + "<br>";
      }
      window.webxdc.setUpdateListener(receiveUpdate, 0);
    </script>
  </body>
</html>
```

Чтобы упаковать приложение в файл `.xdc`, заархивируйте директорию, содержащую `index.html` и все связанные файлы:

```shell
(cd ПУТЬ_К_ДИРЕКТОРИИ && zip -9 --recurse-paths - *) > myapp.xdc
```

Теперь можно отправить файл `myapp.xdc` в любой чат: получатели могут нажать «Запустить», чтобы начать использовать приложение, вводить текст в поле и отправлять обновления друг другу.

Чтобы симулировать участие нескольких человек в чате через браузер, попробуйте [Hello](https://github.com/webxdc/hello) в качестве минимального примера; он содержит всё необходимое для запуска приложения и не требует систем сборки.

[Инструмент симуляции webxdc-dev](https://github.com/webxdc/webxdc-dev) является рекомендуемым инструментом для разработки приложений webxdc, так как он позволяет моделировать работу нескольких пользователей и наблюдать за сетевыми сообщениями между экземплярами приложения. При этом для разработки приложения с помощью `webxdc-dev` мессенджер не требуется.

## Больше примеров

[webxdc на Codeberg](https://codeberg.org/webxdc) и [webxdc на GitHub](https://github.com/webxdc)
содержат подборки примеров приложений webxdc.

В [магазине webxdc](https://webxdc.org/apps) представлены работающие приложения webxdc, которые вы можете использовать уже сегодня. Каждое приложение сопровождается ссылкой на исходный код, чтобы вы могли изучать его и делать ответвления (fork) по своему желанию. Вы можете [предложить свое FOSS-приложение](https://codeberg.org/webxdc/xdcget/src/branch/main/SUBMIT.md) для включения в магазин.

## Полезная информация для разработки

Разработка и развертывание приложений webxdc в целом проще, чем
разработка и поддержка специфичного для приложения HTTP-сервера, работающего в режиме 24/7.
Однако существуют определенные сложности в обеспечении согласованного состояния веб-приложения 
на устройствах пользователей, что является типичной проблемой для любой одноранговой (P2P) сети.
Даже если вы не планируете глубоко изучать эту тему, раздел [Общее состояние веб-приложения](./shared-state/)
познакомит вас с терминологией и важными аспектами, 
уделяя особое внимание webxdc и давая практические рекомендации.
