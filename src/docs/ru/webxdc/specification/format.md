## Формат файла-контейнера Webxdc

-   Приложение Webxdc представляет собой **ZIP-архив** с расширением `.xdc`.
-   ZIP-архив ДОЛЖЕН использовать методы сжатия по умолчанию согласно RFC 1950 — это «Deflate» или «Store».
-   ZIP-архив ДОЛЖЕН содержать как минимум файл `index.html`.
-   ZIP-архив МОЖЕТ содержать файлы `manifest.toml`, `icon.png` или `icon.jpg`.

### Файл manifest.toml

Если ZIP-архив содержит в корне файл `manifest.toml`, из него ДОЛЖНА считываться следующая базовая информация:

```toml
name = "Название моего приложения"
source_code_url = "https://example.org/orga/repo"
```

-   `name` — название приложения webxdc. Если название не задано или манифест отсутствует, в качестве названия используется имя файла.
-   `source_code_url` — опциональный URL, по которому можно найти исходный код приложения и, возможно, другую информацию. Разработчики мессенджеров могут сделать эту ссылку доступной через меню «Справка» в окне приложения.

### Файлы иконок

Если в корне архива присутствует `icon.png` или `icon.jpg`, этот файл используется в качестве иконки приложения. Иконка должна быть квадратной и иметь разумное разрешение, обычно от 128x128 до 512x512 пикселей. Скругление углов, обрезка в круг и прочее добавляются реализациями по мере необходимости, поэтому не стоит добавлять рамки или фигуры в саму иконку. Если иконка не задана, будет использована иконка по умолчанию.
